# AI Guide for Badminton App

This guide helps AI assistants understand the structure and setup of this project.

## Project Overview

This is a full-stack badminton management application with:
- **Frontend**: Vue 3 SPA (Single Page Application) in the `/frontend` directory
- **Backend**: Spring Boot Java application in the `/backend` directory

## Important References

Please refer to the specific guides for detailed instructions:

1. **Frontend Development** - See [Frontend AI Guide](./frontend/.ai-guide.md)
   - Contains instructions on how to run commands on Windows PowerShell
   - Includes npm command syntax specific to this environment
   - Details on troubleshooting common build issues

2. **Backend Development** - See [Backend AI Guide](./backend/.ai-guide.md)
   - Contains instructions for running Maven and Spring Boot
   - Database and configuration setup
   - Backend-specific development and deployment notes

## Project Structure

```
badminton-app/
├── frontend/          # Vue 3 SPA application
├── backend/           # Spring Boot Java backend
├── pom.xml            # Root Maven configuration
├── package.json       # Root npm configuration
└── Dockerfile         # Docker configuration for deployment
```

## Key Points for AI Assistants

- **Shell**: This project runs on Windows PowerShell (not bash)
  - Use `;` to chain commands instead of `&&`
  - Example: `cd frontend; npm run build`

- **Build Tools**:
  - Frontend: npm and Vue CLI
  - Backend: Maven

- **Important Files to Check First**:
  - Frontend: `./frontend/vue.config.js`, `./frontend/package.json`
  - Backend: `./backend/pom.xml`
  - Root: `pom.xml` (parent Maven configuration)

## When Fixing Issues

1. Always check the appropriate guide (frontend or backend)
2. Use PowerShell-compatible command syntax
3. Verify dependency versions match (especially for Vue CLI ecosystem)
4. Use CVE Remediator agent when dealing with dependency vulnerabilities
5. Run builds after making changes to validate fixes

## Common Commands

### Frontend
```powershell
cd E:\learn\badminton-app\frontend
npm install          # Install dependencies
npm run serve        # Development server
npm run build        # Production build
npm run lint         # Run linter
```

### Backend
```powershell
cd E:\learn\badminton-app\backend
mvn clean install    # Build and install
mvn spring-boot:run  # Run the application
```

### Root Level
```powershell
cd E:\learn\badminton-app
npm install          # Install root dependencies
mvn clean install    # Build entire project
```
